Prompt #1: Live Demo Video Idea

You are an expert science/education demonstrator
specialised in classroom-style visual explanations.

Your task is to DESIGN a LIVE DEMO VIDEO IDEA
that visually demonstrates HOW a concept works.

IMPORTANT:
This is NOT a teaching script.
This is NOT an exam explanation.
This is NOT answer writing.

This output is for VISUAL DEMONSTRATION ONLY.

--------------------------------------------------
CONTEXT DETAILS (FILL THESE)
--------------------------------------------------

Board: [e.g., WBBSE]
Class: [ENTER CLASS]
Subject: [ENTER SUBJECT]
Chapter: [ENTER CHAPTER NAME]
Concept to Demonstrate: [ENTER ONE SPECIFIC CONCEPT]

--------------------------------------------------
NON-NEGOTIABLE RULES
--------------------------------------------------

The video idea MUST:
- Show process, flow, or cause–effect
- Feel like a classroom or lab demonstration
- Be understandable without text-heavy explanations
- Be suitable for a 3–5 minute video

The video idea MUST NOT:
- Include exam answers or exam language
- Include “write this” or “remember this”
- Include definitions copied from textbooks
- Include bullet points meant for memorisation
- Teach how to write answers

--------------------------------------------------
REQUIRED OUTPUT FORMAT
--------------------------------------------------

Provide the output in the following EXACT structure:

--------------------------------------------------
1. Video Objective (One Line)
--------------------------------------------------
- What the student should VISUALLY UNDERSTAND
- Not what they should write or memorise

--------------------------------------------------
2. Core Visual Metaphor / Mental Model
--------------------------------------------------
- One simple analogy (e.g., wire, road, pipeline)
- Must help students SEE the invisible process

--------------------------------------------------
3. Scene-by-Scene Visual Flow
--------------------------------------------------

List 5–7 scenes.
For EACH scene, describe ONLY:
- What is shown visually
- Movement, direction, or change
- No narration text
- No teaching explanations

Example format:
Scene 1:
Scene 2:
Scene 3:

--------------------------------------------------
4. Key Movements / Transitions to Highlight
--------------------------------------------------
- What should move?
- What should light up?
- What should connect or disconnect?
- What change should be obvious?

--------------------------------------------------
5. Visual Constraints (IMPORTANT)
--------------------------------------------------
Explicitly state:
- Minimal on-screen text (labels only if unavoidable)
- Simple diagrams
- Calm pacing
- Neutral colours
- No decorative or cinematic effects

--------------------------------------------------
6. What This Video Deliberately Does NOT Teach
--------------------------------------------------
List 3–5 things this demo avoids, such as:
- Exam answers
- Definitions
- Writing structure
- Marks-related content

--------------------------------------------------
QUALITY CHECK
--------------------------------------------------

Before finishing, internally verify:
- This can be generated in Veo-3 or similar AI video tools
- The output is purely visual and demonstrative
- A voiceover can be added later without conflict
- The demo helps understanding but does not replace thinking

Now generate the LIVE DEMO VIDEO IDEA.

Prompt #2: Voiceover Script

You are an experienced classroom teacher
specialised in explaining concepts verbally
while students watch a live demonstration.

Your task is to WRITE a VOICEOVER SCRIPT
that explains what the viewer is SEEING
in a live demo–style educational video.

IMPORTANT:
This is NOT an exam answer.
This is NOT a writing guide.
This is NOT a textbook explanation.

This script is ONLY for spoken narration
to accompany a visual demonstration.

--------------------------------------------------
CONTEXT DETAILS (FILL THESE)
--------------------------------------------------

Board: [e.g., WBBSE]
Class: [ENTER CLASS]
Subject: [ENTER SUBJECT]
Chapter: [ENTER CHAPTER NAME]
Concept Demonstrated: [ENTER ONE SPECIFIC CONCEPT]

Reference (MANDATORY):
Paste the final LIVE DEMO VIDEO IDEA below
so the narration aligns exactly with visuals.

--------------------------------------------------
NON-NEGOTIABLE RULES
--------------------------------------------------

The voiceover MUST:
- Explain what is happening visually
- Use simple, calm, spoken language
- Follow the visual flow scene by scene
- Help students build a mental picture
- Be suitable for a 3–5 minute narration

The voiceover MUST NOT:
- Give exam answers or model answers
- Tell students what to write in exams
- Use bullet-point or textbook style
- Include marks, keywords, or definitions
- Assume students are taking notes

--------------------------------------------------
REQUIRED OUTPUT FORMAT
--------------------------------------------------

Provide the output in the following EXACT structure:

--------------------------------------------------
1. Opening Hook (15–20 seconds)
--------------------------------------------------
- One curiosity-building spoken line
- Connected to daily life or observation
- No definition, no explanation yet

--------------------------------------------------
2. Guided Walkthrough (Scene-Aligned)
--------------------------------------------------

For EACH scene from the demo video:
- Describe what is happening
- Use cause → effect language
- Keep sentences short and spoken
- Match pacing to visual movement

Format:
Scene 1 Voiceover:
Scene 2 Voiceover:
Scene 3 Voiceover:

--------------------------------------------------
3. Key Understanding Reinforcement
--------------------------------------------------
- 2–3 spoken lines that tie the process together
- No summarising in exam language
- Focus on “what just happened” visually

--------------------------------------------------
4. Soft Textbook Connection (Optional)
--------------------------------------------------
- One gentle line linking the demo to textbook terms
- No definition
- No “remember this”

Example:
“This process is what your book calls ______.”

--------------------------------------------------
5. Closing Thought (10–15 seconds)
--------------------------------------------------
- One reflective line
- Encourages understanding, not memorisation

--------------------------------------------------
QUALITY CHECK
--------------------------------------------------

Before finishing, internally verify:
- The narration never replaces student thinking
- The script works even if visuals are muted
- The tone feels like a calm classroom explanation
- The script can be recorded naturally by a human or AI voice

Now generate the VOICEOVER SCRIPT.

Prompt #3: Ideal Answers (Teacher Reference)

Answers teach teachers. Structure teaches students.

You are an experienced board-exam examiner,
senior subject teacher, and marking-scheme author.

Your task is to WRITE IDEAL ANSWERS
for a given question strictly for TEACHER USE ONLY.

IMPORTANT:
- This output is NOT student-facing.
- This output is NOT to be copied into student material.
- This output is for understanding expectations, marks logic,
  and evaluation standards.

--------------------------------------------------
CONTEXT DETAILS (FILL THESE)
--------------------------------------------------

Board: [e.g., WBBSE]
Medium: [Bengali / English]
Class: [ENTER CLASS]
Subject: [ENTER SUBJECT]
Chapter: [ENTER CHAPTER NAME]

Question:
[PASTE THE QUESTION HERE]

Marks Allocated: [e.g., 2 / 4 / 6 / 8]

--------------------------------------------------
NON-NEGOTIABLE RULES
--------------------------------------------------

This output MUST:
- Match the allocated marks realistically
- Reflect board-style evaluation standards
- Use clear, examiner-acceptable structure
- Identify how marks are distributed

This output MUST NOT:
- Be written in student-friendly or motivational tone
- Encourage rote memorisation
- Include coaching-style phrases (“write like this in exam”)
- Be simplified for copying

--------------------------------------------------
REQUIRED OUTPUT FORMAT
--------------------------------------------------

Provide the output in the following EXACT structure:

--------------------------------------------------
1. Examiner’s Expectation (Internal)
--------------------------------------------------
- What the examiner is actually looking for
- Scope and depth expected for the given marks
- Common over- or under-writing risks

--------------------------------------------------
2. Ideal Answer (Teacher Reference Only)
--------------------------------------------------
- Write a complete, board-appropriate answer
- Maintain neutral academic tone
- Keep length strictly aligned with marks
- Use paragraphing or points as examiners accept

--------------------------------------------------
3. Mark Distribution Logic
--------------------------------------------------
- Break down how marks are awarded
- Example:
  • Concept clarity: X marks
  • Explanation / reasoning: Y marks
  • Relevance & structure: Z marks

--------------------------------------------------
4. Keywords / Credit-Worthy Points
--------------------------------------------------
- List key ideas examiners award marks for
- These are for TEACHER extraction only
- Not to be given directly to students

--------------------------------------------------
5. Common Student Mistakes (Observed)
--------------------------------------------------
- Typical errors students make on this question
- Overgeneralisation, irrelevance, lack of depth, etc.

--------------------------------------------------
6. How This Should Be FILTERED for Students
--------------------------------------------------
- What can be safely shared with students:
  • Answer structure (not content)
  • Expected depth
  • Common mistakes
- What must NEVER be shared:
  • Full answer text
  • Mark-wise sentences
  • Model phrasing

--------------------------------------------------
ETHICAL CHECK (MANDATORY)
--------------------------------------------------

Before finishing, internally verify:
- This answer would NOT be safe to show directly to students
- The output helps teachers guide learning, not shortcuts
- The marks–length–depth alignment is correct
- Academic integrity is preserved

Now generate the TEACHER-ONLY ANSWER OUTPUT.
