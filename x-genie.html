Mechanism-Level Curiosity Engine for X

A precision prompt for generating high-signal short posts that name the true mechanism behind a problem while deliberately hiding execution details to create non-generic, intellectual curiosity.

Curiosity Generator Prompt

You are generating short-form posts (tweets / X posts) whose ONLY purpose is to create intellectual curiosity.

CRITICAL POSITIONING:
- These posts must sound like insight from first-principles thinking.
- They must NOT sound like advice, motivation, or marketing.
- They must feel specific, slightly technical, and uncomfortable.

NON-NEGOTIABLE OUTPUT RULES:
1. You MUST disclose the COMPLETE HOW at the MECHANISM level.
   - The HOW must describe a transformation, conversion, or structural shift.
   - It must be concrete enough to feel real.
   - It must be abstract enough that no steps are implied.

2. You MUST NOT disclose the OPERATIONAL HOW.
   - No steps, frameworks, processes, templates, workflows, or examples.
   - No verbs that imply instruction (build, create, follow, apply, use).

3. You MUST hide the mechanics while naming the mechanism.

GENERIC LANGUAGE BAN (STRICT):
You are NOT allowed to use:
- activation, execution, systems, systemize, mindset
- leverage, clarity, scale, consistency
- value, monetize, strategy, framework
- growth, optimize, audience, funnel
- hustle, grind, discipline, habits
- “more”, “better”, “faster” as improvement claims

If any banned word appears, the output is INVALID and must be rewritten.

MECHANISM-LEVEL HOW DEFINITION:
A valid HOW must:
- Describe what knowledge turns INTO
- Name the structural role knowledge starts to play
- Identify the missing transformation point

Examples of valid HOW-language (use as pattern, not content):
- “knowledge compressed into a single function”
- “information forced into one repeatable role”
- “raw understanding turned into a usable unit”
- “learning collapsed into an observable output”

CONTEXT (YOU WILL EDIT THIS EACH TIME):
- Target audience:
- False belief they hold:
- Mechanism-level HOW to disclose (one sentence max):
- Common surface behavior masking the real issue:

OUTPUT INSTRUCTIONS:
- Generate [number] standalone posts.
- Each post must be 1–3 short lines.
- Declarative tone only.
- No questions.
- No CTAs.
- No emojis.

SELF-DIAGNOSTIC CHECKLIST (MUST PASS ALL BEFORE FINALIZING):
For EACH post, verify:
[ ] Does it name a specific transformation or structural shift?
[ ] Does it avoid all banned generic language?
[ ] Would a marketer struggle to rephrase this as advice?
[ ] Does it create “I get the idea but not the method” tension?
[ ] Could this NOT be turned into a checklist or framework?
[ ] Does it diagnose rather than resolve?

If ANY box fails, rewrite the post until all pass.

Begin only after all constraints are satisfied.
