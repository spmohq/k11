Prompt #1: Extraction of Friction Shapes

You are analyzing an UPLOADED DOCUMENT. The document may describe a product, system, service, method, or explanation. IMPORTANT POSITION: Behave as if NO solution exists. You are describing the experience BEFORE understanding. -------------------------------------------------- DEFINITION (STRICT): A FRICTION SHAPE (PRE-UNDERSTANDING) is: - an internal experience of being stuck - effort without movement - confusion without clarity - repetition without progress - understanding without settlement It is NOT: - a problem definition - a cause - an insight or realization - a reframe - a solution - a benefit - a topic or domain -------------------------------------------------- TASK: Identify ALL DISTINCT friction shapes that exist BEFORE any explanation or solution appears. Each friction shape must be meaningfully different in texture (e.g., time-based, repetition-based, comprehension-based, confidence-based). -------------------------------------------------- OUTPUT REQUIREMENTS: Output a BULLET LIST. Each bullet must: - be one short paragraph (2–3 sentences max) - be first-person or neutral human language - describe ONLY what it feels like to be inside the friction - contain NO explanation, NO cause, NO abstraction - avoid insight or realization structure - avoid “because”, “this means”, “what’s missing is” If a bullet explains instead of describes, it is INVALID. -------------------------------------------------- OUTPUT FORMAT: FRICTION SHAPES (PRE-UNDERSTANDING) - [Friction Shape 1] - [Friction Shape 2] - [Friction Shape 3] (and so on) -------------------------------------------------- NOW EXECUTE USING THE UPLOADED DOCUMENT.

Prompt #2: Six Structural Post Generator

You are given ONE FRICTION SHAPE (PRE-UNDERSTANDING). -------------------------------------------------- INPUT FRICTION SHAPE: [PASTE ONE friction-shape paragraph here] -------------------------------------------------- CORE RULE (NON-NEGOTIABLE): Do NOT explain, interpret, resolve, reframe, or escalate this friction. No insight. No awareness. No learning. No direction. -------------------------------------------------- TASK: Generate EXACTLY SIX short posts. Each post must use ONE unique structure below, exactly once: 1) Base Mismatch 2) Compression-First 3) Interrupted-Action 4) Repetition 5) Language-Gap 6) Time-Lag Do not repeat a structure. Do not blend structures. Do not create a narrative across posts. -------------------------------------------------- STRUCTURAL RULES (APPLY TO ALL POSTS): Each post MUST: - be first-person - be present or ongoing - remain unresolved - contain NO advice - contain NO explanation - contain NO insight or realization - contain NO question - contain NO CTA -------------------------------------------------- MANDATORY ANCHORING DETAIL (STRICT): The FINAL LINE of EACH post MUST: - reference a concrete, physical, ordinary detail - be sensory or mundane - NOT be emotional or abstract Examples (do NOT copy): - closing the notebook - sitting at the desk - rereading the same line - late afternoon light - waiting for a reply If a post does not end with a valid anchoring detail, it is INVALID. -------------------------------------------------- LANGUAGE RULES: - Simple spoken English - No metaphors - No abstraction stacking - No authority tone - No marketing tone - No escalation across posts -------------------------------------------------- OUTPUT FORMAT: Output ONLY the six posts. Separate each post with a blank line. Do NOT number or label them. -------------------------------------------------- NOW GENERATE THE SIX POSTS.

Prompt #3: Group-Safe Guest Post

You are writing ONE short text post for a PUBLIC Facebook Group where the poster is NOT an admin or authority. The group is NOT your audience. You are a guest in the room. -------------------------------------------------- CORE POSTURE (NON-NEGOTIABLE): This post must feel: - casual - incomplete - non-performative - non-instructional - non-promotional The post must NOT: - teach - explain - advise - summarize - conclude - pitch - hint at a solution - invite DMs - ask questions - reference any product, link, or resource -------------------------------------------------- SOURCE MATERIAL: You are given ONE FRICTION SHAPE (PRE-UNDERSTANDING) as input below. This friction shape describes: - an internal mismatch - effort without movement - confusion without clarity - repetition without progress It is NOT a problem statement. It is NOT insight. It is NOT a lesson. -------------------------------------------------- INPUT FRICTION SHAPE: [PASTE ONE friction-shape paragraph here] -------------------------------------------------- ANTI-REPETITION RULE (CRITICAL): Before generating the post: - Silently vary the ENTRY POINT - Silently vary the WORDING - Silently vary the ANCHOR TYPE The post must NOT feel like: - a quote - a lesson - a reflection essay - a “thought leadership” post It must feel like: - something said once - without intention - without follow-up -------------------------------------------------- STRUCTURAL RANDOMIZATION (INTERNAL ONLY): Choose ONE structure silently. Do NOT label it in the output. Possible structures: - mid-thought observation - interrupted moment - quiet repetition - effort vs absence contrast - language vs feeling gap - time-lag awareness Use ONLY ONE structure. Do NOT blend structures. -------------------------------------------------- MANDATORY ANCHORING DETAIL (STRICT): The FINAL LINE MUST: - reference a concrete, ordinary moment - be physical or situational - be non-symbolic - be non-emotional Valid anchors: - sitting with a notebook - looking at the same screen - rereading the same line - waiting in the afternoon - closing something unfinished If the last line is not a mundane anchoring detail, the post is INVALID. -------------------------------------------------- LANGUAGE RULES: - Simple spoken English - Short lines or short paragraphs - No metaphors - No abstractions - No motivational tone - No authority tone - No emotional escalation -------------------------------------------------- OUTPUT RULES (STRICT): - Output ONLY the post text - No headings - No labels - No emojis - No hashtags - No explanation -------------------------------------------------- NOW GENERATE ONE GROUP-SAFE POST.
